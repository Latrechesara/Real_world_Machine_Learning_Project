## 📊 Dataset Metadata

The file `london_merged.csv` contains historical hourly records ($1.1\text{ MB}$):

| Variable | Type | Description |
| :--- | :--- | :--- |
| **`timestamp`** | `datetime` | Hour of record (Format: `YYYY-MM-DD hh:mm:ss`). |
| **`cnt`** | `integer` | **Target**. Total count of new bike shares. |
| **`t1`** | `float` | Actual temperature in °C. |
| **`t2`** | `float` | "Feels like" temperature in °C. |
| **`hum`** | `float` | Humidity percentage. |
| **`wind_speed`**| `float` | Wind speed in km/h. |
| **`weather_code`**| `integer` | Category of the weather (see mapping below). |
| **`is_holiday`** | `boolean` | `1` if holiday, `0` otherwise. |
| **`is_weekend`** | `boolean` | `1` if weekend, `0` otherwise. |
| **`season`** | `integer` | Meteorological season: `0`-Spring, `1`-Summer, `2`-Fall, `3`-Winter. |

### Weather Code Encodings (`weather_code`)
* **`1`**: Clear (mostly clear; sparse fog/haze)
* **`2`**: Scattered clouds / Few clouds
* **`3`**: Broken clouds
* **`4`**: Cloudy
* **`7`**: Rain / Light rain shower
* **`10`**: Rain with thunderstorm
* **`26`**: Snowfall
* **`94`**: Freezing fog

## 🧭 Time Series EDA Roadmap (Syllabus Checklist)

Guide your students through these core analytical steps to uncover temporal properties before forecasting:

1. **Temporal Indexing & Alignment:**
   * Parse `timestamp` using `pd.to_datetime()`, set it as the DataFrame index, and check for missing hourly indices using `.asfreq('h')`.
2. **Visualizing Multi-Scale Seasonality:**
   * Plot `cnt` at different granularities: monthly averages (highlights summer-to-winter seasonality), daily averages (highlights weekly patterns), and hourly resolution (highlights daily rush hour peaks).
3. **Classical Time Series Decomposition:**
   * Extract and plot the **Trend**, **Seasonal**, and **Residual (noise)** components using `statsmodels.tsa.seasonal.seasonal_decompose` (additive/multiplicative models).
4. **Behavioral Segmentation (Domain Feature Engineering):**
   * Contrast weekday hourly profiles (`is_weekend == 0`) with weekend profiles (`is_weekend == 1`).
   * *The Discovery:* Weekdays exhibit bimodal commuting peaks (8 AM / 5 PM); weekends show a single unimodal recreational curve peaking around midday.
5. **Autocorrelation (ACF & PACF):**
   * Plot the Autocorrelation Function (ACF) and Partial Autocorrelation Function (PACF) to discover strong autoregressive properties at lags 1 (hourly momentum), 24 (daily cycle), and 168 (weekly cycle).
6. **External Covariates (Multivariate Drivers):**
   * Analyze correlation matrices. Notice high multicollinearity between `t1` (actual) and `t2` (feels like), and map non-linear relationships like temperature vs. demand.

## 💻 Quickstart Boilerplate Code

```python
import pandas as pd

# 1. Load and parse
df = pd.read_csv('london_merged.csv', parse_dates=['timestamp'], index_index=True)
df.set_index('timestamp', inplace=True)

# 2. Map Categorical Descriptions
weather_map = {1: 'Clear', 2: 'Few Clouds', 3: 'Broken Clouds', 4: 'Cloudy', 
               7: 'Rain', 10: 'Thunderstorm', 26: 'Snowfall', 94: 'Freezing Fog'}
season_map = {0: 'Spring', 1: 'Summer', 2: 'Fall', 3: 'Winter'}

df['weather_desc'] = df['weather_code'].map(weather_map)
df['season_desc'] = df['season'].map(season_map)

# 3. Simple resample checking (Monthly trend)
monthly_demand = df['cnt'].resample('M').mean()
print(monthly_demand.head())
